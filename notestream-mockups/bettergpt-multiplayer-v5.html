What "share" actually does

Promoting a local chat to a shared workspace does three concrete things:

  1. Snapshots the conversation — every turn so far is mirrored to a cloud workspace. Local session keeps running, cloud is the new source of truth for invited users.
  2. Projects the context you picked — files mounted read-only into the cloud sandbox; MCPs you didn't tick stay on your machine.
  3. Routes new prompts through the cloud agent — when you or a teammate prompts, the cloud session responds with the shared context. Your local session pauses.

What it doesn't do

  • Doesn't give teammates terminal access on your machine
  • Doesn't share your unticked MCPs (Linear, Gmail, etc.)
  • Doesn't permanently copy your codebase — read-only mount, revoke anytime
🔒 Revoke at any time. Removes cloud access; conversation history stays in your local session.

08 Decision · build inside betterGPT vs new repo Two parallel paths · we're doing both

Path A · inside betterGPT

primary

Continue on bettergpt/prototype-core. Add Story 6 (group/shared workspaces) as the next milestone after solo polish ships. One product, one brand, sequential roadmap.

Pros

  • Reuses 30+ commits of prototype momentum
  • Single brand · bettergpt.ai already owned
  • User-stories doc already includes group/sharing in Story 6
  • Solo demos validate the loop before adding complexity

Cons

  • Inherits Cortex tech debt
  • Multiplayer is bolted onto solo-first design

Path B · new project

parallel

Stand up ~/code/shared-chats/ as a separate repo. Multiplayer-first architecture. Fresh choices (Postgres, no god file, Cloud Run on existing GCP). Vendor Cortex's ClaudeSessionEngine.

Pros

  • Clean architecture · no inherited debt
  • Multiplayer-first product · different positioning
  • Independent shipping cadence
  • Two-product portfolio (solo + team)

Cons

  • Two codebases to maintain
  • Brand fragmentation if not careful
  • Cortex modules to vendor (3-file lift)
Doing both in parallel. Path A (betterGPT) is the path to a real product fastest. Path B (new project) is a hedge — if betterGPT's solo-first frame blocks multiplayer cleanly, Path B is ready to absorb the design. The mockup above applies to both — same UX, different host. The architectural decision can defer; the design doesn't have to.
